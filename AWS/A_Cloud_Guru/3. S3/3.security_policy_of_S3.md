## üîê S3 Bucket Default Security Settings

### 1Ô∏è‚É£ Bucket is Private üîí

- The bucket is **private by default** and cannot be accessed by anyone unless explicitly allowed.

### 2Ô∏è‚É£ Only the Owner Can Access It üë§

- The **AWS account that created the bucket has full access**.
- No **IAM users, groups, or other AWS accounts** can access it unless permissions are granted.

### 3Ô∏è‚É£ No Public Access üö´

- **Public access is completely blocked by default**.
- The bucket **cannot be accessed via a public URL** unless explicitly allowed.

### 4Ô∏è‚É£ Encryption Is Not Enabled by Default üîê

- Server-side encryption **is not enabled** automatically.
- You must manually enable encryption using **SSE-S3, SSE-KMS, or SSE-C** if needed.

## üìú S3 Bucket Policy

### üîπ Key Components

1Ô∏è‚É£ **Effect** ‚Äì Specifies whether access is `"Allow"` or `"Deny"`.

- `"Allow"` ‚Äì Grants access to the specified actions.
- `"Deny"` ‚Äì Denies access to the specified actions.

2Ô∏è‚É£ **Principal** ‚Äì Defines **who** can access the bucket (IAM users, roles, AWS accounts, or services).

- `"*"` ‚Äì Allows access to everyone (public access).
- `"AWS": "arn:aws:iam::123456789012:root"` ‚Äì Allows access to a specific AWS account.
- `"Service": "s3.amazonaws.com"` ‚Äì Allows access to a specific AWS service.
- `"AWS": ["arn:aws:iam::123456789012:user/User1", "arn:aws:iam::123456789012:role/Role1"]` ‚Äì Allows access to specific IAM users or roles.

3Ô∏è‚É£ **Action** ‚Äì Lists the **permitted actions**, such as `s3:GetObject` or `s3:PutObject`.

- **Read Actions:**

  - `s3:GetObject` ‚Äì Read objects from the bucket.
  - `s3:ListBucket` ‚Äì List objects in the bucket.

- **Write Actions:**

  - `s3:PutObject` ‚Äì Upload objects to the bucket.
  - `s3:DeleteObject` ‚Äì Delete objects from the bucket.

- **Bucket Management:**

  - `s3:CreateBucket` ‚Äì Create a new S3 bucket.
  - `s3:DeleteBucket` ‚Äì Delete an existing S3 bucket.
  - `s3:PutBucketPolicy` ‚Äì Apply or modify the bucket policy.

- **Permissions & Security:**
  - `s3:GetBucketPolicy` ‚Äì View the bucket policy.
  - `s3:PutBucketAcl` ‚Äì Modify bucket ACL (Access Control List).
  - `s3:GetObjectAcl` ‚Äì Get object ACL settings.

4Ô∏è‚É£ **Resource** ‚Äì Defines the **bucket or object(s)** to which the policy applies.

- `"arn:aws:s3:::your-bucket-name"` ‚Äì Applies to the bucket itself.
- `"arn:aws:s3:::your-bucket-name/*"` ‚Äì Applies to all objects inside the bucket.
- `"arn:aws:s3:::your-bucket-name/folder-name/*"` ‚Äì Applies to all objects inside a specific folder.

### üîê Example S3 Bucket Policy

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::your-bucket-name/*"
    }
  ]
}
```

## üìú S3 ACLs (Access Control Lists)

S3 ACLs provide **object-level access control**, deciding **who can access a specific object** and **what actions they can perform** (read, write, full control).

`same as Bucket policy but it's object level`
