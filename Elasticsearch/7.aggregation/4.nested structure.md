## **üìå Sub-Aggregations Elasticsearch Me Kya Hote Hain?**

Elasticsearch me **sub-aggregations** ka matlab hai ki ek aggregation ke andar **dusri aggregations chalana**. Yeh **grouped data pe multiple calculations perform** karne me madad karte hain.

---

## **1Ô∏è‚É£ Nested Aggregation Structure Ka Example (Sub-Aggregations)**

Yeh example **ek bucket aggregation (`terms`) ko use karta hai aur uske andar multiple metric aggregations (`sum`, `avg`, `max`) run karta hai**.

### **‚úÖ Example Query**

```json
GET index_name/_search
{
  "size": 0,
  "aggs": {
    "status_terms": {   // <-- Bucket Aggregation (terms)
      "terms": { "field": "status" },
      "aggs": {
        "total_sales": { "sum": { "field": "total_amount" } },
        "avg_sales": { "avg": { "field": "total_amount" } },
        "max_sales": { "max": { "field": "total_amount" } }
      }
    }
  }
}
```

### **üìå Yeh Query Kya Kar Rahi Hai?**

1Ô∏è‚É£ **Documents ko `status` ke basis pe group kar rahi hai** (bucket aggregation).  
2Ô∏è‚É£ **Har `status` ke andar `total_amount` ka `sum`, `avg` aur `max` calculate kar rahi hai**.

---

### **üìå Expected Output Example**

```json
{
  "aggregations": {
    "status_terms": {
      "buckets": [
        {
          "key": "completed",
          "doc_count": 500,
          "total_sales": { "value": 625250.0 },
          "avg_sales": { "value": 1250.5 },
          "max_sales": { "value": 5000.75 }
        },
        {
          "key": "pending",
          "doc_count": 200,
          "total_sales": { "value": 178150.0 },
          "avg_sales": { "value": 890.75 },
          "max_sales": { "value": 3000.25 }
        }
      ]
    }
  }
}
```

üìå **Yeh output dikhata hai ki har `status` ke liye total sales, average sales aur max sales kya hain.**

## **2Ô∏è‚É£ filter aggregation Ke Sath Sub-Aggregations Ka Use**

Agar tumhe pehle **documents filter karne hain** aur fir uss filtered data pe sub-aggregations chalani hain, toh tum **filter aggregations** use kar sakte ho.

### **‚úÖ Example Query with a `filter` Aggregation**

```json
GET /order/default/_search
{
  "size": 0,
  "aggs": {
    "low_value": {
      "filter": {
        "range": {
          "total_amount": {
            "lt": 50
          }
        }
      },
      "aggs": {
        "avg_amount": {
          "avg": {
            "field": "total_amount"
          }
        }
      }
    }
  }
}
```

### **üìå Yeh Query Kya Kar Rahi Hai?**

1Ô∏è‚É£ **Filter aggregation (`low_value`) sirf un documents ko select karti hai jisme `total_amount < 50` hai.**  
2Ô∏è‚É£ Is **filtered data ke andar `avg` sub-aggregation (`avg_amount`) total amount ka average calculate karti hai.**

üìå **Isse yeh ensure hota hai ki aggregation calculations sirf filtered documents pe chalein.**

## üî¢ remember one thing üòé

> **`this sub aggregation not return any documents`**

```json
GET /order/default/_search
{
  "query": {
    "bool": {
      "must": [
        { "range": { "total_amount": { "lt": 50 } } },
        { "term": { "status": "pending" } }
      ]
    }
  },
  "aggs": {
    "avg_amount": { "avg": { "field": "total_amount" } }
  }
}
```

> **`this sub aggregation return all documents total amount is less than 50 and status is pending`**

## **3Ô∏è‚É£ Example: Filters Aggregation with Multiple Filters (Status + Total Amount)**

This query creates **separate buckets** for different combinations of `status` and `total_amount` conditions.

```json
GET index_name/_search
{
  "size": 0,
  "aggs": {
    "order_filters": {
      "filters": {
        "filters": {
          "completed_high_value_orders": {
            "bool": {
              "must": [
                { "term": { "status": "completed" } },
                { "range": { "total_amount": { "gte": 500 } } }
              ]
            }
          },
          "pending_low_value_orders": {
            "bool": {
              "must": [
                { "term": { "status": "pending" } },
                { "range": { "total_amount": { "lt": 100 } } }
              ]
            }
          }
        }
      },
      "aggs": {
        "total_sales": {
          "sum": { "field": "total_amount" }
        }
      }
    }
  }
}
```

---

### **üìå Expected Output Example:**

```json
{
  "aggregations": {
    "order_filters": {
      "buckets": {
        "completed_high_value_orders": {
          "doc_count": 200,
          "total_sales": { "value": 125000.0 }
        },
        "pending_low_value_orders": {
          "doc_count": 150,
          "total_sales": { "value": 7500.0 }
        }
      }
    }
  }
}
```
